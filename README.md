# Java Sorting Strategy App

Учебный командный проект для реализации алгоритмов сортировки кастомных объектов с использованием паттернов проектирования.

## 🛠 Технологии
- Java 17
- Gradle

## 📦 Функционал
- Сортировка кастомных объектов
- Паттерн Strategy (разные алгоритмы сортировки)
- Паттерн Builder (создание объектов)
- Валидация данных
- Варианты ввода:
    - вручную
    - случайная генерация
    - загрузка из файла

## 🧠 Архитектура
Проект построен с разделением ответственности:
- модели (models)
- стратегии сортировки (strategy)
- сервисы (service)
- ввод/вывод (io)

## 🚀 Запуск
```bash
./gradlew run
```
## 👥 Git Workflow

### 📌 Работа с задачами
1. Возьмите задачу с канбан-доски  
2. Назначьте её на себя  
3. Переведите статус в **In Progress**

Если задачи нет - создайте новую и обсудите с командой

---

### 🌿 Ветки

Каждая задача выполняется в отдельной ветке, созданной от `main`

Формат именования веток:

- feature/jssa-<номер-задачи>
- bugfix/jssa-<номер-задачи>
- hotfix/jssa-<номер-задачи>

Примеры:

- feature/jssa-1
- feature/jssa-3
- bugfix/jssa-2
---

### 🔁 Работа с кодом

Перед созданием Pull Request:

1. Обновите свою ветку от `main`:
```bash
git pull --rebase origin main
```
Разрешите конфликты (если есть)

🔍 Pull Request
- В main напрямую не пушим
- Все изменения идут через Pull Request
- Требуется минимум 1 approval
- После ревью задача переводится в Done

🔄 Процесс
- Берем задачу
- Создаем ветку
- Реализуем функционал
- Делаем PR
- Проходим ревью
- Мержим в main
- Берем следующую задачу
