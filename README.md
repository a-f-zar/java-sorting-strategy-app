# Java Sorting Strategy App

Учебный командный проект на Java по реализации алгоритмов сортировки кастомных объектов с использованием паттернов `Strategy` и `Builder`.

В качестве доменной модели выбран класс `Student`.

## Описание проекта

Приложение работает в консольном режиме и позволяет:

- заполнять коллекцию студентов вручную;
- генерировать случайную коллекцию студентов;
- загружать коллекцию студентов из JSON-файла;
- выбирать алгоритм сортировки;
- выбирать поле сортировки;
- сортировать студентов по всем 3 полям;
- выполнять дополнительную сортировку только по четным значениям номера зачетной книжки;
- подсчитывать количество вхождений заданного студента в коллекцию многопоточным способом;
- сохранять отсортированную коллекцию и результат поиска в файл в режиме добавления данных.

Программа работает в цикле и завершает работу только при выборе соответствующего пункта меню.

## Доменная модель

В проекте используется кастомный класс `Student` со следующими полями:

- `name`
- `averageGrade`
- `studentCardNumber`

Для создания объектов реализован паттерн `Builder`.

## Реализованный функционал

### Основное задание

- реализован класс `Student`;
- реализован `Builder` для `Student`;
- переопределены `toString()`, `equals()` и `hashCode()`;
- реализована валидация входных данных;
- реализованы компараторы для сортировки по всем 3 полям;
- реализован паттерн `Strategy` для выбора алгоритма сортировки;
- реализованы алгоритмы сортировки:
  - `BubbleSort`
  - `InsertionSort`
  - `SelectionSort`
- реализован ввод данных:
  - вручную через консоль
  - случайная генерация
  - загрузка из JSON-файла
- реализовано консольное меню и основной цикл приложения;
- добавлены тесты для основных компонентов приложения.

### Дополнительные задания

- реализована сортировка объектов по числовому полю, где сортируются только элементы с четными значениями, а элементы с нечетными значениями остаются на исходных позициях;
- реализована запись отсортированных коллекций и найденных значений в файл в режиме добавления данных;
- заполнение коллекций случайными данными и загрузка из файла выполнены с использованием Stream API;
- реализован многопоточный метод подсчета количества вхождений студента в коллекцию.

## Структура проекта

Основные пакеты проекта:

- `com.aston.models` — доменные модели
- `com.aston.models.comparator` — компараторы для сортировки
- `com.aston.sorting` — алгоритмы сортировки и контекст сортировки
- `com.aston.input` — парсинг и загрузка входных данных
- `com.aston.validation` — валидация данных
- `com.aston.output` — запись результатов в файл
- `com.aston.concurrent` — многопоточная логика подсчета вхождений
- `com.aston.ui` — консольное меню приложения

## Формат файла для загрузки

Для загрузки коллекции используется JSON-массив объектов.

Пример:

```json
[
  {
    "name": "Anna",
    "averageGrade": 4.8,
    "studentCardNumber": 12345
  },
  {
    "name": "Roman",
    "averageGrade": 3.9,
    "studentCardNumber": 67890
  }
]
```

Пример файла также находится в проекте:

- `src/main/resources/students.json`

## Особенности реализации

- для сортировки не используются готовые реализации сортировок;
- выбор алгоритма сортировки реализован через паттерн `Strategy`;
- создание объекта `Student` реализовано через паттерн `Builder`;
- для всех вводимых данных реализована валидация;
- загрузка из файла пропускает невалидные записи;
- запись результатов в файл выполняется в режиме добавления данных;
- многопоточный подсчет вхождений выполняется через пул потоков.
